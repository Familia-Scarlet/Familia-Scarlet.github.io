# 大数据概念

海量、高增长率和多样化的信息资产

数据储存单位是TB、PB、EB以上

大数据主要解决海量数据的==采集==、==存储==和==分析计算==问题

# 大数据特点

4个V

  1. Volume

    海量数据

  2. Velocity

    数据产生快，比如天猫双十一96s内，交易额达到100亿

  3. Variety

    数据类型多，有结构化数据和非结构化数据，以数据库、JSON文档代表的结构化数据，以网络日志、音视频、图片、地理信息为代表的非结构化数据

  4. Value

    价值密度的高低与数据总量的大小成反比，数据量庞大但价值可能不高。所以，需要进行数据清洗，将无效数据、低价值数据过滤掉。

# 大数据应用场景

  1. 爱好分析，如B站、淘宝等平台

  2. 分析用户消费习惯，经典案例，纸尿布+啤酒

  3. 物流仓储，在对某产品高需求的地方多备货，根据物流量配相应数量的货车数量，帮助企业进行决策

  4. 保险、金融、房产、贷款，分析用户的消费、还款能力

  5. 炼丹，ChA、深度学习都需要大量数据模型支持

# Hadoop概述

Hadoop是一个有Apache基金会所开发的分布式系统基础架构

主要解决，海量数据的储存和海量数据的分析计算问题

广义上，Hadoop通常是指一个更广泛的概念——Hadoop生态圈

# Hadoop发展历史

  1. 创始人Doug Cutting，起初只是为了实现与Google类似的全文搜索功能，在Lucene框架基础上进行优化升级查询引擎和索引引擎
  2. 2001年年年Lucene成为Apache基金会的子项目
  3. Lucene和Google都碰到海量数据的场景
  4. 参考Google的解决方案，实现微型Nutch
  5. 依据Google在大数据方面的三篇论文
    • GFS->HDFS
    • Map-Reduce->MR
    • BigTable->HBase
  6. 2005，Hadoop作为Lucene子项目被引入Apache
  7. 2006，MapReduce和NDFS被纳入Hadoop项目，Hadoop正式诞生

# Hadoop的三大发行版本

  • Apache

    最基础版本

  • Cloudera

    内部继承很多大数据框架，2008发行产品CDH（Hadoop的云产品）

  • Hortonworks

    同样对Apache的Hadoop进行封装，推出HDP，后被Cloudera收购，推出新的品牌CDP

# Hadoop优势

  • 高可靠

    Hadoop底层维护多个数据副本，一份数据在多个集群中重复保存

  • 高扩展性

    在集群间分配任务数据，动态增缩集群

  • 高效性

    MapReduce思想下，Hadoop并行工作以加速任务处理，这里的并行不是多线程多进程，而是多集群

  • 高容错性

    能将失败的任务重新分配
